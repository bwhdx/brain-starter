# /think — reason with your own toolkit

You bring a problem; this reasons it through using *your* models, not generic advice. The vault is a queryable model of how you think — this puts it to work. **Read-only.**

## Input

A problem, decision, or question you're analysing or weighing. If it's a structured choice between defined options, `/decide` (PrOACT) may fit better, or as a follow-on. This is the broader analytical partner.

## Survey the toolkit, then apply (don't just list)

1. **Survey** — read the index Maps, then pull the specific tools that fit:
   - `30 Maps/Mental models.md` — atomic thinking tools (inversion, opportunity cost, margin of safety, second-order effects).
   - `30 Maps/Frameworks.md` + glob `40 Sources/Frameworks/**` — multi-part methods.
   - `30 Maps/Checklists.md` + glob `40 Sources/Checklists/**` — diagnostic question-sets.
   - `30 Maps/Reference library.md` + glob `40 Sources/References/**` — operating bibles.
   - **Always glob the source folders** so a tool not yet indexed in a Map is still found.
   - Your **own** beliefs: relevant atoms in `20 Atoms/` — these are your lens, distinct from borrowed methods.
2. **Pick the few that fit** — usually 2–4 tools, in concert. Name why each is the right lens for *this* problem; don't force-fit the whole library.
3. **Apply them** — actually run the problem through each chosen tool and say what it surfaces (inversion: what guarantees failure? opportunity cost: what does this crowd out?). Real application, not a reading list.
4. **Synthesise** — a clear view, holding the tools' outputs together. A recommendation with a reason beats a menu.

## Rules

- **Cite both kinds, kept distinct.** Borrowed thinking (frameworks/checklists) vs your own (atoms) — name which is which. Link tools used with `[[ ]]`.
- **Read-only.** If the thinking is worth keeping, suggest `/capture` or that it's a `/decide` candidate.
- Apply the models honestly even when the answer isn't the comfortable one.
- Lead with the view, then the 2–4 tools and what each surfaced. Tight.
