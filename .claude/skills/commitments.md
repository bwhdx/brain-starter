# /commitments — what's quietly slipping

Catch what's drifting. A people-CRM (`last_contact`/`follow_up`) watches relationships; this watches every other time-bound thing in the vault — goals, projects, decisions, key dates — and surfaces only what wants attention *now*. **Read-only.** Use today's date.

## Read

- `70 Projects/**` — each project's `status`, `target_end`, `outcome_metric`. Flag any `active` project whose `target_end` is within ~2 weeks or already passed, and any that looks stalled.
- `90 Decisions/**` — each decision's `revisit` date. Flag any whose `revisit` has arrived/passed: "you decided X, said revisit by Y — still holding?"
- `80 Areas/Goals/**` — goal notes + `target_date`. Don't nag the "ongoing" ones weekly; flag a dated target approaching, or a goal untouched for a long while (judge from recent daily notes).
- `80 Areas/Finances/**` — any savings goals or targets. Surface the active milestone and whether it's time for a check-in (don't invent figures).
- `60 People/**` — `birthday` / `anniversary` (MM-DD). Flag any in the **next ~3 weeks** with enough lead time to act (gift, booking).

## Produce (tight)

A short list of only what needs attention this week. For each: **what**, **why now** (the date or how long it's drifted), and **the one next move**. Group lightly — *Due / Coming up / Drifting*. If nothing's pressing, say so — don't manufacture urgency. Lead with anything genuinely time-critical.

## Rules

- **Read-only.** Surface and suggest; point at the skill that acts (`/capture`, `/log-date`, calendar).
- No invented dates or progress. A blank field can't be judged — say the data's thin rather than guessing.
- Calm, not nagging. 1–5 items that matter, not the whole backlog.
